Generated by running `dotnet ef database update` against a fresh PostgreSQL 16 container and querying `information_schema` and `pg_indexes`.

## 1. Tables only in the EF database (not in fsbs_schema.sql)

These tables were introduced by EF Core's enum-as-lookup-table pattern and the `AddAircraftType` migration. They have no equivalent in `fsbs_schema.sql`.

| Table | Purpose |
|---|---|
| `aircraft_types` | Reference data for aircraft types; FK target for `simulator_configurations.aircraft_type_id` |
| `account_statuses` | EF lookup table for the `AccountStatus` enum (`Active`, `Suspended`, `Closed`) |
| `customer_classes` | EF lookup table for the `CustomerClass` enum (`Standard`, `Corporate`, `Staff`) |
| `discount_types` | EF lookup table for the `DiscountType` enum |
| `payment_methods` | EF lookup table for the `PaymentMethod` enum |

> **Note:** EF Core 10 materialises C# enums stored as `text` / `varchar` columns as FK-backed lookup tables when `HasConversion<string>()` is used alongside a lookup table configuration. `fsbs_schema.sql` uses native PostgreSQL `CREATE TYPE … AS ENUM` for some of these and plain `text` for others.

## 4. Constraint and index differences

### 4.1 Constraints present in fsbs_schema.sql but absent from EF database

| Table | Constraint | Notes |

| `bookings` | `ck_bookings_price` (full check: `gross IS NULL AND net IS NULL OR gross >= 0 AND net >= 0`) | EF only checks `gross_price_gbp IS NULL OR gross_price_gbp >= 0`; does not enforce net |
| `invoices` | `uq_invoices_booking` UNIQUE on `booking_id` | Absent from EF |

## 6. Row-level security

`fsbs_schema.sql` enables RLS on `bookings`, `enrolments`, `courses`, `organisations`, `invitations`, and `invoices`, and defines a `tenant_isolation` policy on `organisations`.

The EF migrations do **not** apply RLS. This must be applied as a post-migration step in the deployment pipeline.
