# FSBS.Infrastructure.Persistence

The EF Core persistence core. Owns the `DbContext`, the audit interceptor, and the DI registration extension that wires everything together. All other persistence sub-projects depend on this one.

## Responsibilities

- **`FsbsDbContext`**: 34 `DbSet<T>` properties, `HasDefaultSchema("fsbs")`, delegates all Fluent API configuration to `FSBS.Infrastructure.Persistence.Entities` via `ApplyConfigurationsFromAssembly`
- **Global query filters**: registered in `FsbsDbContext.ApplyGlobalQueryFilters`
  - Soft-delete (`!e.IsDeleted`) applied to every `ISoftDeletable` entity
  - Tenant isolation (`e.TenantId == currentUser.TenantId`) applied to `AppUser`, `Organisation`, and `Course`; evaluated per-request from the injected `ICurrentUser`
- **`AuditInterceptor`**: `SaveChangesInterceptor` that stamps `CreatedAt`, `CreatedBy`, `UpdatedAt`, `UpdatedBy` on every `AuditableEntity` before `SaveChanges` completes; registered as a singleton
- **`PersistenceServiceExtensions.AddPersistence`**: registers the context with Npgsql, `UseSnakeCaseNamingConvention`, the audit interceptor, and the migrations assembly pointer

## Key configuration decisions

| Concern | Choice |
|---|---|
| Column naming | `UseSnakeCaseNamingConvention()` (EFCore.NamingConventions) |
| Schema | `fsbs` — never `public` |
| PKs | `uuid`, generated by `uuid_generate_v4()` database default |
| Timestamps | `timestamptz` / C# `DateTimeOffset` |
| Optimistic concurrency | PostgreSQL `xmin` system column (`xid` type), configured per entity |
| Migrations assembly | `FSBS.Infrastructure.Persistence.Migrations` (referenced as a string to avoid a circular project dependency) |

## Dependencies

```
FSBS.Application  (ICurrentUser)
FSBS.Infrastructure.Persistence.Entities  (configuration assembly anchor)
Npgsql.EntityFrameworkCore.PostgreSQL 10.0.1
EFCore.NamingConventions 10.0.1
Microsoft.EntityFrameworkCore.Relational 10.0.7
```

## Do not add

- `IEntityTypeConfiguration<T>` files (those belong in `FSBS.Infrastructure.Persistence.Entities`)
- EF migrations or `IDesignTimeDbContextFactory` (those belong in `FSBS.Infrastructure.Persistence.Migrations`)
- Repository implementations (those belong in `FSBS.Infrastructure.Persistence.Repositories`)
- Business logic or domain rules
- `Microsoft.EntityFrameworkCore.Design` — keeping it out of this project prevents accidental `dotnet ef` invocations against the wrong startup project
